# 01. 문서구조

## DOCTYPE
- 브라우저에게 HTML 버전을 알려주는 선언
- HTML5는 `<!DOCTYPE html>` 하나로 통일
- 없으면 브라우저가 quirks mode(호환 모드)로 동작 → 렌더링이 달라질 수 있음
- quirks mode: 브라우저마다 다르게 렌더링 → 레이아웃 깨짐 발생 가능

## 기본 구조
```html
<!DOCTYPE html>
<html lang="ko">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>페이지 제목</title>
  </head>
  <body>
    <!-- 화면에 보이는 내용 -->
  </body>
</html>
```

## 각 태그 역할
| 태그 | 역할 |
|------|------|
| `<html>` | 문서 전체를 감싸는 루트 요소, lang 속성으로 언어 지정 |
| `<head>` | 브라우저에게 필요한 정보 (화면에 안 보임) |
| `<body>` | 실제 화면에 보이는 내용 |
| `<meta charset>` | 문자 인코딩 설정 (UTF-8 = 한글 포함 다국어 지원) |
| `<meta viewport>` | 모바일 화면 대응 (없으면 모바일에서 축소되어 보임) |
| `<title>` | 브라우저 탭에 표시되는 제목, SEO에도 영향 |

## head vs body
- `head` → 브라우저/검색엔진을 위한 정보 (CSS, JS 링크, 메타정보 등)
- `body` → 사용자에게 보여지는 내용

## lang 속성
- 스크린리더가 언어를 인식해 올바른 발음으로 읽어줌
- 검색엔진이 언어별로 검색 결과를 분류할 때 사용
- `ko` = 한국어, `en` = 영어, `ja` = 일본어

## meta charset
- UTF-8을 안 쓰면 한글이 깨질 수 있음
- 반드시 `<head>` 가장 첫 번째에 선언 권장
  ```html
  <head>
    <meta charset="UTF-8"> <!-- 가장 먼저 -->
    ...
  </head>
  ```

## meta viewport
```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```
- `width=device-width` → 페이지 너비를 디바이스 화면 너비에 맞춤
- `initial-scale=1.0` → 초기 줌 레벨 1배 (확대/축소 없음)
- 없으면 모바일에서 PC 화면을 축소해서 보여줌

## head 안에 들어오는 것들
```html
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <meta name="description" content="페이지 설명">
  <title>제목</title>
  <link rel="stylesheet" href="style.css">  <!-- CSS -->
  <link rel="icon" href="/favicon.ico">     <!-- 파비콘 -->
  <script src="app.js" defer></script>      <!-- JS -->
</head>
```
