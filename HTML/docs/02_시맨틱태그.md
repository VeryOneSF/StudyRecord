# 02. 시맨틱태그

## 시맨틱이란?
- 태그 자체가 **의미**를 가지는 것
- `<div>` 대신 의미있는 태그를 써서 코드의 역할을 명확하게 함
- 검색엔진(SEO), 접근성, 유지보수에 유리

## div vs 시맨틱 태그
```html
<!-- div만 쓴 경우 - 의미 없음 -->
<div class="header">...</div>
<div class="nav">...</div>
<div class="content">...</div>

<!-- 시맨틱 태그 - 의미 있음 -->
<header>...</header>
<nav>...</nav>
<main>...</main>
```

## 페이지 전체 구조 예시
```html
<body>
  <header>
    <nav>...</nav>
  </header>

  <main>
    <section>
      <h2>섹션 제목</h2>
      <article>...</article>
      <article>...</article>
    </section>
    <aside>사이드바</aside>
  </main>

  <footer>...</footer>
</body>
```

## 주요 시맨틱 태그
| 태그 | 역할 | 주의사항 |
|------|------|------|
| `<header>` | 페이지 또는 섹션의 머리말 | 페이지에 여러 개 가능 |
| `<nav>` | 네비게이션 링크 모음 | 주요 링크 모음에만 사용 |
| `<main>` | 페이지의 핵심 콘텐츠 | 페이지당 반드시 1개 |
| `<section>` | 주제별로 묶인 콘텐츠 구역 | 제목(h태그)이 있어야 함 |
| `<article>` | 독립적으로 배포 가능한 콘텐츠 | 단독으로 의미가 있어야 함 |
| `<aside>` | 본문과 간접적으로 관련된 내용 | 사이드바, 광고, 관련 링크 |
| `<footer>` | 페이지 또는 섹션의 바닥글 | 페이지에 여러 개 가능 |

## section vs article
- `section` → 페이지 안에서 주제별로 묶는 용도, 단독으로는 의미 없음
- `article` → 그 자체로 독립적인 콘텐츠 (RSS로 배포해도 말이 됨)
```html
<section>
  <h2>최신 글</h2>
  <article>
    <h3>글 제목 1</h3>
    <p>내용...</p>
  </article>
  <article>
    <h3>글 제목 2</h3>
    <p>내용...</p>
  </article>
</section>
```

## 텍스트 관련 태그
| 태그 | 역할 | 비고 |
|------|------|------|
| `<h1>~<h6>` | 제목 | h1은 페이지당 1개 권장, 순서 지켜야 함 |
| `<p>` | 단락 | 블록 요소 |
| `<span>` | 인라인 텍스트 묶음 | 의미 없음, 스타일 적용용 |
| `<strong>` | 중요한 텍스트 | 굵게 + 의미 있음 (스크린리더가 강조) |
| `<em>` | 강조 텍스트 | 기울임 + 의미 있음 |
| `<br>` | 줄바꿈 | 남용 금지, 여백은 CSS로 |
| `<hr>` | 수평선 (주제 구분) | 시각적 구분이 아닌 의미적 구분 |
| `<blockquote>` | 인용문 | `cite` 속성으로 출처 명시 |
| `<code>` | 인라인 코드 | 코드 한 줄 |
| `<pre>` | 서식 유지 텍스트 | 줄바꿈, 공백 그대로 표시 |

## 블록 vs 인라인
- **블록 요소** → 혼자 한 줄 차지 (`div`, `p`, `h1`, `section` 등)
- **인라인 요소** → 텍스트 흐름 안에 위치 (`span`, `a`, `strong`, `em` 등)
