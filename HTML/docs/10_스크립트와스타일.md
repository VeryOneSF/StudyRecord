# 10. 스크립트와 스타일

## CSS 연결
```html
<!-- 외부 파일 (권장) -->
<link rel="stylesheet" href="style.css">

<!-- 인라인 스타일 시트 (전역 스타일 일부에 사용) -->
<style>
  body { margin: 0; }
</style>

<!-- 인라인 스타일 (권장하지 않음 - 유지보수 어려움) -->
<p style="color: red;">텍스트</p>
```

## CSS 우선순위
```
인라인 스타일 > <style> 태그 > 외부 CSS 파일
```
- 인라인 스타일은 우선순위가 높아서 덮어쓰기 어려움 → 남용 금지

## JS 연결
```html
<!-- 외부 파일 (권장) -->
<script src="app.js" defer></script>

<!-- 인라인 스크립트 -->
<script>
  console.log("hello");
</script>

<!-- 모듈 방식 -->
<script type="module" src="app.js"></script>
```

## type="module"
- ES6 모듈 사용 가능 (`import`, `export`)
- 기본적으로 `defer`처럼 동작
- 같은 파일을 여러 번 import해도 한 번만 실행
- 자동으로 strict mode 적용

## link 태그 주요 rel 값
| rel 값 | 역할 |
|------|------|
| `stylesheet` | CSS 파일 연결 |
| `icon` | 파비콘 |
| `preload` | 리소스 미리 로드 |
| `prefetch` | 다음 페이지 리소스 미리 로드 |
| `preconnect` | 외부 도메인 미리 연결 |
| `canonical` | 대표 URL 지정 |
| `alternate` | 대체 페이지 (다국어, RSS 등) |

## 파비콘
```html
<link rel="icon" href="/favicon.ico">
<link rel="icon" href="/favicon.png" type="image/png">
<link rel="apple-touch-icon" href="/apple-touch-icon.png"> <!-- iOS -->
```

## 폰트 연결
```html
<!-- Google Fonts -->
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=Noto+Sans+KR&display=swap" rel="stylesheet">

<!-- 로컬 폰트 -->
<style>
  @font-face {
    font-family: 'MyFont';
    src: url('/fonts/myfont.woff2') format('woff2');
  }
</style>
```

## 위치에 따른 차이
```html
<head>
  <link rel="stylesheet" href="style.css"> <!-- CSS는 head에 (FOUC 방지) -->
</head>
<body>
  ...
  <script src="app.js" defer></script> <!-- JS는 defer 또는 body 끝 -->
</body>
```
- CSS를 head에 두는 이유 → 렌더링 전에 스타일 적용, FOUC(스타일 없는 화면) 방지
- JS를 defer 하는 이유 → HTML 파싱을 막지 않기 위해
