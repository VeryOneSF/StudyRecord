# 06. 접근성 (Accessibility)

## 접근성이란?
- 장애인, 고령자 등 모든 사용자가 웹을 사용할 수 있도록 하는 것
- 스크린리더, 키보드만으로 탐색 가능하게 만드는 것
- UI 프레임워크 개발 시 특히 중요
- WCAG(Web Content Accessibility Guidelines) 가이드라인을 기준으로 함

## alt 속성
```html
<!-- 의미있는 이미지 -->
<img src="logo.png" alt="회사 로고">

<!-- 장식용 이미지 (스크린리더가 무시) -->
<img src="divider.png" alt="">

<!-- alt 아예 없으면 파일명을 읽어버림 → 반드시 alt 작성 -->
```

## aria 속성
| 속성 | 역할 |
|------|------|
| `aria-label` | 요소의 이름을 직접 지정 |
| `aria-labelledby` | 다른 요소의 텍스트를 이름으로 참조 |
| `aria-describedby` | 추가 설명을 다른 요소에서 참조 |
| `aria-hidden` | 스크린리더에서 숨김 |
| `aria-expanded` | 열림/닫힘 상태 (드롭다운, 아코디언) |
| `aria-disabled` | 비활성화 상태 |
| `aria-required` | 필수 입력 |
| `aria-invalid` | 유효하지 않은 입력 |
| `aria-live` | 동적으로 변하는 영역 알림 |

```html
<!-- 아이콘 버튼 (텍스트 없을 때) -->
<button aria-label="메뉴 열기">
  <svg>...</svg>
</button>

<!-- 드롭다운 상태 표시 -->
<button aria-expanded="false" aria-controls="menu">메뉴</button>
<ul id="menu">...</ul>

<!-- 에러 메시지 연결 -->
<input aria-describedby="error-msg">
<p id="error-msg" aria-live="polite">이메일 형식이 올바르지 않습니다.</p>
```

## role 속성
- 시맨틱하지 않은 요소에 의미를 부여
```html
<div role="button" tabindex="0">클릭</div>
<div role="dialog" aria-modal="true">모달</div>
<div role="alert">에러 메시지</div>
<div role="navigation">네비게이션</div>
<div role="tab">탭</div>
<div role="tabpanel">탭 패널</div>
```
- 가능하면 시맨틱 태그 사용이 우선, role은 커스텀 컴포넌트에서 주로 사용

## 키보드 접근성
```html
<!-- tabindex: 키보드 탭 순서 지정 -->
<div tabindex="0">포커스 가능 (탭 순서에 포함)</div>
<div tabindex="-1">포커스 가능하지만 탭 순서 제외 (JS로 포커스)</div>
<div tabindex="1">탭 순서 직접 지정 (권장하지 않음)</div>
```
- 키보드 탭 순서는 DOM 순서를 따르는 게 원칙
- `tabindex="1"` 이상 직접 지정은 혼란을 줄 수 있어 비권장

## 시맨틱 태그 vs aria
```html
<!-- 이게 최선 - 시맨틱 태그가 자동으로 접근성 제공 -->
<button>클릭</button>

<!-- 이건 aria 직접 추가해야 함 -->
<div role="button" tabindex="0" aria-label="클릭">클릭</div>
```
- 시맨틱 태그를 쓰면 aria 없이도 접근성이 올라감
- `<button>` → 키보드(Enter, Space), 스크린리더 자동 지원
- `<div onclick>` → 직접 다 추가해야 함

## 색상 대비
- 텍스트와 배경 색상의 대비율이 충분해야 함
- WCAG AA 기준: 일반 텍스트 4.5:1 이상, 큰 텍스트 3:1 이상
- 색상만으로 정보를 전달하면 안 됨 (색맹 사용자 고려)
