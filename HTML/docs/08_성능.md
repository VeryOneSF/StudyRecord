# 08. 성능 (Performance)

## 스크립트 로딩 방식
```html
<!-- 일반: HTML 파싱 멈추고 JS 로드 + 실행 후 재개 -->
<script src="app.js"></script>

<!-- defer: HTML 파싱 유지, 파싱 완료 후 순서대로 실행 -->
<script src="app.js" defer></script>

<!-- async: HTML 파싱 유지, 로드 완료 즉시 실행 (순서 보장 안 됨) -->
<script src="app.js" async></script>
```

## defer vs async
| | 일반 | defer | async |
|---|---|---|---|
| HTML 파싱 | 멈춤 | 유지 | 유지 |
| 실행 시점 | 즉시 | 파싱 완료 후 | 로드 완료 즉시 |
| 실행 순서 | 보장 | 보장 | 보장 안 됨 |
| DOM 접근 | 위험 | 안전 | 위험할 수 있음 |
| 용도 | - | 일반적인 JS | 독립 스크립트 (광고, 분석) |

## 이미지 최적화
```html
<!-- lazy loading: 뷰포트에 들어올 때 로드 -->
<img src="image.jpg" alt="설명" loading="lazy">

<!-- eager: 즉시 로드 (기본값, 첫 화면 이미지에 사용) -->
<img src="hero.jpg" alt="설명" loading="eager">
```

## preload / prefetch / preconnect
```html
<!-- preload: 현재 페이지에서 곧 필요한 리소스 미리 로드 -->
<link rel="preload" href="font.woff2" as="font" crossorigin>
<link rel="preload" href="hero.jpg" as="image">
<link rel="preload" href="critical.css" as="style">

<!-- prefetch: 다음 페이지에서 필요할 리소스 미리 로드 (낮은 우선순위) -->
<link rel="prefetch" href="next-page.js">

<!-- preconnect: 외부 도메인에 미리 연결 (DNS + TCP + TLS) -->
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="dns-prefetch" href="https://fonts.googleapis.com"> <!-- DNS만 미리 -->
```

## Core Web Vitals
- 구글이 SEO에 반영하는 성능 지표
| 지표 | 의미 | 기준 |
|------|------|------|
| LCP (Largest Contentful Paint) | 가장 큰 콘텐츠 로드 시간 | 2.5초 이하 |
| FID (First Input Delay) | 첫 입력 응답 지연 | 100ms 이하 |
| CLS (Cumulative Layout Shift) | 레이아웃 밀림 정도 | 0.1 이하 |

## 렌더링 차단 리소스
```html
<!-- CSS는 head에 → 렌더링 전에 스타일 적용 (FOUC 방지) -->
<head>
  <link rel="stylesheet" href="style.css">
</head>

<!-- JS는 defer 또는 body 끝에 → HTML 파싱 막지 않기 위해 -->
<body>
  ...
  <script src="app.js" defer></script>
</body>
```
- FOUC(Flash of Unstyled Content): CSS 로드 전에 스타일 없는 화면이 잠깐 보이는 현상
