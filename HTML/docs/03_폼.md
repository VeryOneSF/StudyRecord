# 03. 폼 (Form)

## 기본 구조
```html
<form action="/submit" method="POST">
  <label for="username">이름</label>
  <input id="username" type="text" name="username" placeholder="이름 입력">
  <button type="submit">제출</button>
</form>
```

## form 속성
| 속성 | 역할 |
|------|------|
| `action` | 데이터를 전송할 URL |
| `method` | GET / POST |
| `enctype` | 파일 업로드 시 `multipart/form-data` 필요 |
| `novalidate` | 브라우저 기본 유효성 검사 비활성화 |
| `autocomplete` | 자동완성 on/off |

## GET vs POST
| | GET | POST |
|---|---|---|
| 데이터 위치 | URL 쿼리스트링 | body |
| 보안 | 낮음 (URL에 노출) | 높음 |
| 용도 | 검색, 조회 | 로그인, 회원가입, 파일 업로드 |
| 길이 제한 | 있음 | 없음 |

## input type 종류
| type | 용도 |
|------|------|
| `text` | 일반 텍스트 |
| `password` | 비밀번호 (입력값 숨김) |
| `email` | 이메일 형식 자동 검증 |
| `number` | 숫자 입력 |
| `tel` | 전화번호 |
| `url` | URL 형식 검증 |
| `checkbox` | 다중 선택 |
| `radio` | 단일 선택 (같은 name끼리 묶임) |
| `file` | 파일 업로드 |
| `date` | 날짜 선택 |
| `range` | 슬라이더 |
| `color` | 색상 선택 |
| `hidden` | 화면에 안 보이는 값 전송 |
| `submit` | 폼 제출 버튼 |
| `reset` | 폼 초기화 버튼 |
| `search` | 검색창 |

## 유효성 검사 속성
| 속성 | 역할 |
|------|------|
| `required` | 필수 입력 |
| `minlength` / `maxlength` | 최소/최대 글자 수 |
| `min` / `max` | 최소/최대 값 |
| `pattern` | 정규식으로 형식 검증 |
| `disabled` | 비활성화 (값 전송 안 됨) |
| `readonly` | 읽기 전용 (값 전송 됨) |

```html
<!-- 패턴 예시: 한국 전화번호 -->
<input type="tel" pattern="[0-9]{2,3}-[0-9]{3,4}-[0-9]{4}">
```

## 주요 폼 요소
```html
<!-- 드롭다운 -->
<select name="color">
  <option value="">선택하세요</option>
  <option value="red" selected>빨강</option>
  <option value="blue">파랑</option>
</select>

<!-- 여러 줄 텍스트 -->
<textarea name="content" rows="5" cols="30"></textarea>

<!-- 체크박스 -->
<input type="checkbox" id="agree" name="agree" value="yes">
<label for="agree">동의합니다</label>

<!-- 라디오 버튼 (같은 name으로 묶음) -->
<input type="radio" id="male" name="gender" value="male">
<label for="male">남성</label>
<input type="radio" id="female" name="gender" value="female">
<label for="female">여성</label>

<!-- 파일 업로드 -->
<input type="file" accept=".jpg,.png" multiple>
```

## label 중요성
- `for`와 `id`를 연결하면 label 클릭 시 input에 포커스됨
- 접근성(스크린리더)에 필수
- label 없는 input은 스크린리더가 뭔지 알 수 없음

## button type
```html
<button type="submit">제출</button>  <!-- 폼 제출 -->
<button type="reset">초기화</button> <!-- 폼 초기화 -->
<button type="button">그냥 버튼</button> <!-- 폼과 무관 -->
```
- `type` 생략 시 기본값이 `submit` → form 안에서 의도치 않게 제출될 수 있음
- form 안에 버튼 쓸 때 `type="button"` 명시 권장

## fieldset / legend
```html
<!-- 관련 입력 요소 그룹화 -->
<fieldset>
  <legend>개인정보</legend>
  <label for="name">이름</label>
  <input id="name" type="text">
</fieldset>
```
